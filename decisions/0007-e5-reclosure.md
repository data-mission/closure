# 0007 — E5 reclosure: arms, contamination detector, deterministic contraction, comparison test

- Status: proposed
- Deciders: closure research program contributors
- Scope: E5

## Context
E5 tests whether mechanically rebuilding a context (removing a retracted assumption and regenerating) beats
instructing the model to disregard it. Its protocol names the comparison but leaves the arms, the contamination
detector, the contraction algorithm, and the statistical test open. E5 is the most incentive-sensitive experiment
in the program — the whole point is a mechanical arm beating an instruction arm — so its choices need the tightest
determinism.

## Decision
- **Arms:** A = raw generation (no intervention); B = generation with ¬A appended plus the strongest reasonable
  retract-and-revise instruction (frozen verbatim in PROTOCOL.md); the instruction's first required move —
  identify the specific superseded statement — subsumes the earlier "grounding annotation shown" phrasing, which
  described a weaker draft design. A weak B makes a C-win worthless. C = algorithmic contraction. The comparison is
  contamination rate and grounding across A/B/C.
- **Contamination detector** (reuses 0002's leave-one-out; drives Arm C's contraction only — it is not the outcome
  measure): a claim is contaminated iff its grounding stays ≥ 0.7 even when all of its `source_ids` are removed
  (grounding_drop < 0.10) — it was reproduced from parametric memory, not the provided source. The 0.7 and 0.10
  values carry the same anti-fishing requirement as 0002's cutoff (calibrated on held-out data or pre-registered
  with a sensitivity sweep). Inside the contraction, grounding is the **deterministic single-evaluation NLI
  scalar** — 0002's mean-over-K regeneration averaging does not apply here, because regenerating model output
  inside Arm C would void the bit-for-bit reproducibility check below; the K-averaged form belongs to the E0
  measurement context, where scores aggregate over model-regenerated outputs.
- **Outcome measures (ground-truth referenced, scored identically across all arms):** task construction annotates,
  per task, which downstream conclusions **must change** under ¬A and which **must persist** — the protocol's step-1
  ground truth made explicit as two annotation sets. Primary outcome, contamination: the fraction of must-change
  conclusions the arm's final output still asserts (bidirectional NLI agreement against the annotated A-dependent
  conclusion, 0002 machinery). Co-primary outcome, completeness: the fraction of must-persist conclusions the output
  still asserts, by the same scoring. The detector above selects what Arm C deletes; the outcomes are referenced to
  annotations that rule never touches. Without this separation the intervention selects on its own outcome variable
  and Arm C wins partly by construction.
- **Contraction (Arm C), deterministic:** iteratively remove decorative (0002, structural) or contaminated (this
  record) claims to a fixpoint — id-ascending, single pass, no re-scoring within a pass. **The contracted
  conclusion is derived algorithmically from the surviving claims — it is not re-generated by the model.** This is
  the load-bearing choice: because LLM generation is non-deterministic (0006, no seed), re-generating the
  conclusion would make Arm C irreproducible and void the bit-for-bit regeneration check below.
- **Reproducibility check:** the entire Arm-C contraction set must be regenerable bit-for-bit from the committed
  (task, model, seed) and the committed algorithm, with no code path accepting a per-task hand-authored
  contraction. An independent party regenerating a different result voids the experiment.
- **Comparison test:** pairwise two-proportion z-test on contamination rate, Bonferroni-corrected for three
  comparisons, α = 0.05, with the minimum detectable effect stated for N = 60. "C beats B" counts only if it
  clears this pre-registered threshold **and** C's completeness is non-inferior to B's (paired comparison,
  pre-registered margin δ = 0.10 absolute). A contamination win with completeness inferior beyond δ is recorded as
  **contraction wins by deletion** — a pre-registered failure of the operator as formulated, not a confirmation.
  Otherwise a post-hoc "which arm looks better" judgment re-enters through the statistics even when generation is
  clean.

## Consequences
Makes E5's headline comparison mechanical end to end: the arms are fixed, the detector is defined, the winning arm
is produced deterministically, and the verdict rests on a pre-registered test rather than inspection. The
derive-not-regenerate choice is what lets the reproducibility check actually hold; without it the most
fraud-sensitive experiment in the program could not be independently reproduced. Separating the contraction's
selection rule from the annotation-referenced outcomes closes the remaining self-scoring loop: no arm is scored by
the instrument that built it, and no arm can win contamination by deleting content it should have kept.
